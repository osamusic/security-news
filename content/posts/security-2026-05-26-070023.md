### [Oncology Institute Discloses Data Breach](https://www.securityweek.com/oncology-institute-discloses-third-party-data-breach/)
**2026-05-25 21:17 JST ／ SecurityWeek ／ MEDIUM**


**要約:** The Oncology Institute (TOI) が、2025年に第三者ベンダー（TriZetto Provider Solutions の可能性が高い）で発生したサイバーインシデントに起因するデータ侵害を確認した。100以上のクリニックから成るネットワークの患者データに不正アクセスがあり、漏洩情報には保護対象保健情報（PHI）が含まれる。フォレンジック企業 Kroll の関与とタイムラインから、ベンダーレベルで数百万人に影響する大規模侵害の一部とみられる。


**なぜ重要か:** 医療機関は自前のセキュリティが堅牢でも、決済・請求処理を担う第三者ベンダー経由で大量のPHIを失いうる。本件はサプライチェーン上流の侵害が下流の多数の医療機関へ波及する典型例である。


**影響:** TOI 傘下100以上のクリニックの患者が影響を受け、PHIの不正利用・なりすまし・標的型詐欺のリスクにさらされる。通知までの長いリードタイムも被害拡大の一因となる。


**次のアクション:** 自組織が利用する請求・決済系ベンダー（特に TriZetto/Cognizant 系）の利用状況を棚卸しし、ベンダー側の侵害通知体制と契約上の責任分界を確認する。患者・利用者向けにはクレジットモニタリングやなりすまし対策の案内を準備する。

**要約:** Radiology Associates of Richmond (RAR) が、2025年7月のインシデントに起因し266,183人に影響するデータ侵害を公表した。2026年4月に完了したフォレンジック調査で、攻撃者が内部システムへ不正アクセスし、機微なPHIを含むファイルを窃取したことが確認された。漏洩データには氏名、社会保障番号、政府発行ID、金融口座情報、詳細な医療・健康保険記録が含まれる。


**なぜ重要か:** 画像診断クリニックという比較的小規模な医療機関でも、内部システム侵入により26万人規模・SSNや金融情報を含む高機微データが流出しうることを示す。発生から公表まで約10か月を要しており、検知・対応の遅さも課題である。


**影響:** 流出データはなりすまし・金融詐欺・医療詐欺に直結する組み合わせであり、被害者は長期的なリスクを負う。医療セクター全体での同種攻撃の常態化を裏付ける事例でもある。


**次のアクション:** 医療機関は内部システムへの不正アクセス検知（EDR/監査ログ）とインシデント対応のリードタイム短縮を見直す。PHIを含むファイルサーバへのアクセス制御と暗号化、最小権限の徹底を確認する。

### [Ghost CMS CVE-2026-26980 Exploited to Hijack 700+ Sites for ClickFix Attacks](https://thehackernews.com/2026/05/ghost-cms-cve-2026-26980-exploited-to.html)
**2026-05-25 21:02 JST ／ The Hacker News ／ MEDIUM**


**要約:** Ghost CMS の Content API に存在する深刻なSQLインジェクション脆弱性 CVE-2026-26980（CVSS 9.4）を悪用し、700以上のサイトが乗っ取られた。攻撃者は本脆弱性で認証不要のまま Admin API キーを窃取し、サイトコンテンツに悪意あるJavaScriptを注入。偽CAPTCHAを表示してユーザーにPowerShellコマンドを実行させる「ClickFix」攻撃を誘発し、認証情報窃取マルウェアを配布する。大学・ブロックチェーン・フィンテック分野が影響を受けた。


**なぜ重要か:** 認証不要のAPI脆弱性が管理者キーの奪取に直結し、さらに ClickFix というソーシャルエンジニアリングと組み合わさることで、サイト訪問者にまで被害が連鎖する多段攻撃となっている。CVSS 9.4 かつ実環境での大規模悪用が確認済みで、緊急性が高い。


**影響:** Ghost CMS を運用するサイトは、コンテンツ改ざん・管理者権限奪取・訪問者へのマルウェア配布の踏み台化のリスクを負う。フィンテック・ブロックチェーン系では二次被害が大きい。


**次のアクション:** Ghost CMS を CVE-2026-26980 修正版へ即時アップデートする。Admin API キーをローテーションし、コンテンツに不審なJavaScriptが注入されていないか確認する。Content API へのアクセスログを点検し、不正アクセスの痕跡を調査する。

**なぜ重要か:** 高度に自動化された手法で数千プロジェクトのCI/CDパイプラインに同時かつ持続的なアクセスを維持する点が脅威的である。窃取対象がクラウド資格情報やKubernetes設定に及ぶため、単一の侵害が組織のクラウド基盤全体へ波及しうる。
